## github 查看某文件每次提交的变化情况

这里分 **网页端 GitHub** 和 **本地 Git 命令行** 两种常用方式，查看单个文件历次提交、逐版改动。

# 一、GitHub 网页端（在线查看，无需本地环境）
## 1. 查看文件所有提交记录（commits）
1. 打开仓库 → 找到目标文件，点击进入文件页
2. 页面右上角点击 **`History`**（历史）
   - 会列出**所有修改过该文件的提交**，按时间倒序展示每条 commit 标题、作者、时间、哈希值。

## 2. 查看单条提交里该文件的改动
- 点任意一条 commit 条目 → 进入提交详情页
- 页面只看当前文件的**增删代码/文字**，标绿新增、标红删除。

## 3. 对比「任意两个版本」差异（重点）
1. 在文件 History 页，找到两个不同提交
2. 勾选两条记录，或点击版本旁的 **`...` → Compare**
3. 即可横向对比该文件两个历史版本的完整差异。

## 快捷直达链接格式
把下面链接替换参数可直接访问文件历史：
```
https://github.com/用户名/仓库名/commits/分支名/文件路径
```
示例：
```
https://github.com/xxx/test-repo/commits/main/src/app.js
```

# 二、本地 Git 命令行（功能最全，推荐开发者）
进入仓库根目录执行。

## 1. 查看该文件的**所有提交日志**
```bash
git log --oneline 文件名
```
- `--oneline` 精简输出，只显示 commit ID + 提交说明
- 不带参数会显示完整作者、时间、备注

示例：
```bash
git log --oneline README.md
```

## 2. 查看「每次提交 + 对应代码改动」（逐版看 diff）
```bash
git log -p 文件名
```
- `-p` = patch，直接展示每一次提交对该文件的**详细增删**

## 3. 只看精简改动（统计行数）
```bash
git log --stat 文件名
```

## 4. 对比「当前文件」和「某个历史版本」
先用上面命令拿到 commit 短哈希（如 `a1b2c3`）：
```bash
git diff a1b2c3 HEAD 文件名
```
- `HEAD` 代表当前最新版本

## 5. 对比**两个历史版本**之间该文件差异
```bash
git diff 旧commitID 新commitID 文件名
```

## 6. 图形化浏览（可选，直观）
如果装了 `gitk` / `git gui`：
```bash
gitk 文件名
```
弹出图形界面，点左侧提交，右侧实时看代码变动。

# 三、常用简写 & 技巧
1. 只看最近 N 条记录（例如最近10条）
```bash
git log -p -10 文件名
```
2. 按作者筛选该文件提交
```bash
git log --author="用户名" 文件名
```
