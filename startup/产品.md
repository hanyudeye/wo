# 一、先把问题拆清楚（关键）

你现在的约束本质是：

* 工作成长性差（技能不复利）
* 市场竞争力不突出（跳槽困难）
* 时间被工作消耗（学习空间有限）

# 二、优先级最高的方向（不要分散）

## 第一目标：提升“可卖技能”

不是学很多东西，而是选一个方向做到能换工作：

### 推荐三条现实路线（选一条）

#### 1. 后端 / 全栈（最稳）

你已经学 Node.js，这条是合理的：

重点能力：

* API 设计
* 数据库（MySQL / PostgreSQL）
* Redis 基础
* 项目部署（Docker）

目标：

> 能独立做一个可上线的项目

#### 2. AI应用开发（更高溢价）

现在市场需求更强：

重点：

* LLM API（OpenAI / Claude）
* RAG（向量数据库）
* Agent workflow
* 简单模型调用 + 产品化

目标：

> 做“AI工具类产品”

# 四、第三目标：增加“非工资收入”

## 1. 工具产品（最适合你）

例如你之前提的：

* Windows清理工具
* 自动化工具
* 小插件

模式：

* 免费 + 高级版收费
* 或一次性付费

---

## 2. 短视频知识变现

你可以做：

* 电脑效率工具
* AI工具使用
* Windows优化
* 自动化脚本

重点不是“教技术”，而是：

> “解决具体痛点”

# 五、最关键的现实策略（很多人忽略）

## 不要同时做很多方向

你现在最容易犯的错误：

```text id="1"
学习 + 副业 + 投资 + 换工作 + 杂项
```

结果：

> 每一项都半吊子

---

## 正确方式：

只保留一个主线：

### 主线 = 技术能力提升（能跳槽）

副线：

* 一个副业实验（小工具或短视频）

---

# 六、收入提升的本质公式

你要记住这个结构：

> 收入 = 市场价值 × 可替代性 × 议价能力

你现在的问题是：

* 市场价值低（岗位限制）
* 可替代性高（技能浅）
* 议价能力弱（跳槽困难）

所以要做的是：

---

## 1. 提高市场价值

## 2. 降低可替代性

→ 做项目 / 作品 / 产品

## 3. 提高议价能力

→ 能跳槽才有工资上涨空间
